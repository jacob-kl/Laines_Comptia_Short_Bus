Blockchain Hashing — Why It's Immutable

Edit a block's data and watch it break every subsequent block. This is how blockchain uses hashing for integrity — not encryption.

Exam takeaway: Blockchain uses SHA-256 for immutability. Each block stores the previous block's hash. Change any block → its hash changes → the next block's "previous hash" no longer matches → chain broken → tampering detected. Integrity mechanism, not confidentiality.
Zero Trust Architecture — Control & Data Plane

Tap each component to understand its role. The Panel below updates on tap/click.

Exam shortcut: PE = brain (decides). PA = messenger (issues/revokes tokens). PEP = gatekeeper (allows/blocks traffic). Control plane = PE + PA. Data plane = PEP. Adaptive identity = trust adjusts dynamically.
PKI — Chain of Trust

Tap each node to understand its role in the hierarchy.

Exam shortcut: Root CA = offline, self-signed, trust anchor. Intermediate CA = online, issues end-entity certs. Chain verification: cert → intermediate → root in trust store = VALID.

PKI Components
ComponentDescription
Root CASelf-signed. Trust anchor. Must be OFFLINE/air-gapped. Signs only intermediate CAs. Compromise = entire PKI destroyed.
Intermediate CASigned by root. Online. Issues end-entity certs. Compromise isolated from root.
CertificateBinds identity to public key. Contains: subject, SAN, validity, key usage. Signed by CA.
CSREntity generates keypair, sends public key + identity info to CA. Private key NEVER sent.
CRLRevocation list. Downloaded periodically. Can be stale between publications.
OCSPReal-time per-cert revocation check. Better freshness than CRL.
OCSP StaplingServer caches OCSP response, presents in TLS handshake. Removes OCSP server dependency.
Wildcard cert*.domain.com — one compromise exposes all subdomains.
SAN certLists specific FQDNs. More granular — compromise limited to listed domains.
Zero Trust Components
ComponentPlaneRole
Policy Engine (PE)ControlBrain. Evaluates access request vs policy, risk score, device health, context. Makes grant/deny decision.
Policy Admin (PA)ControlIssues/revokes session tokens based on PE decision. Communicates with PEP.
Policy Enf. Point (PEP)DataGatekeeper. Physically allows or blocks traffic based on PA signal.
Adaptive Identity—Trust score changes dynamically with risk signals. Anomaly → step-up auth or session revocation.
Micro-segmentationDataGranular east-west workload control. No implicit trust between servers on same network.
