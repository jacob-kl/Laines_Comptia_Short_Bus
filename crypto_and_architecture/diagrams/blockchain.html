Why This Creates Immutability

Each block hashes its own content PLUS the previous block's hash. Change any block → its hash changes → the next block's "previous hash" no longer matches → every block after breaks. In a distributed network, a tampered chain is rejected by consensus.
1
Each block hashes its content. SHA-256 produces a unique digest. Any change — even one character — produces a completely different hash.
2
Each block stores the previous block's hash. This is how the chain is formed — each block cryptographically references the one before it.
3
Changing Block N cascades forward. Block N's hash changes → Block N+1's "prev hash" is now wrong → Block N+1's own hash changes → Block N+2 breaks → cascade.
4
Exam takeaway: Blockchain uses hashing for integrity, not confidentiality. SHA-256 is the hash function. The immutability comes from the chained structure — not encryption.
