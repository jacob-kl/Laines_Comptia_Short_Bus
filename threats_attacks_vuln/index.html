Cyber Kill Chain — Attack Lifecycle

The 7 phases of an attack from reconnaissance to objective. Tap each phase to see what attackers do and how defenders can stop it at that stage.

Exam relevance: The kill chain concept shows WHY defense in depth matters — stopping an attacker at any phase prevents the attack from progressing. Defenders don't need to stop every phase; stopping one is sufficient.
Attack Type Comparison — Tap to Compare

The exam frequently presents scenarios where you must distinguish between similar attack types. Tap each card for the exam-critical differentiators.

Key differentiators the exam loves: Phishing vs watering hole (email vs compromised site). Single vs double extortion ransomware. SQLi vs XSS (database vs browser). Virus vs worm (user action vs autonomous). Stored vs reflected XSS (persistent vs URL-based).

AttackHow It WorksKey Differentiator

VishingVoice/phone-based phishingPHONE — not email
SmishingSMS-based phishing with malicious linkTEXT MESSAGE — not email
BECSpoof/compromise exec email for wire fraudFinancial fraud, executive impersonation
PretextingFabricate scenario to extract infoMade-up backstory/identity
Watering holeCompromise trusted site frequented by targetsAttacks a third-party site, not victim directly
TyposquattingLookalike domain/package name for typospaypa1.com vs paypal.com
Malware Types
TypeBehaviorDefining Characteristic
VirusAttaches to host file, spreads via user actionRequires HOST FILE + USER ACTION to spread
WormSelf-replicates across network autonomouslyNO user action needed — exploits vulnerabilities
TrojanDisguised as legitimate software, hidden payloadAppears legitimate — user installs voluntarily
RATRemote access/control of infected systemGives attacker INTERACTIVE remote control
RootkitHides at OS/kernel level, intercepts sys callsHIDES malware from AV/EDR at kernel level
KeyloggerRecords keystrokes for credential theftCaptures credentials WITHOUT needing network access
RansomwareEncrypts files, demands ransomENCRYPTION + extortion; double = also exfiltrates
SpywareSilently collects user dataSurveillance — screen, mic, webcam, history
Botnet/C2Network of compromised machines under attacker controlUsed for DDoS, spam, credential stuffing

Application Attack Mitigations
AttackRoot CausePrimary Mitigation
SQL InjectionUser input concatenated into SQL queryParameterized queries (prepared statements)
XSSUser input rendered as HTML/JS without encodingOutput encoding + Content Security Policy (CSP)
Buffer OverflowNo bounds checking on input to memory bufferASLR + DEP/NX + stack canaries + safe functions
CSRFBrowser sends cookies automatically with requestsCSRF tokens + SameSite cookie attribute
