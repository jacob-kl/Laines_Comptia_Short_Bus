PKI Components
ComponentRole
Root CAApex of trust chain. Offline in most orgs. Signs intermediate CAs.
Intermediate CAIssues end-entity certificates. Online. Compromise isolated from root.
CertificateBinds public key to identity. Signed by CA. Contains subject, validity, SAN, key usage.
CSRCertificate Signing Request. Entity generates keypair, sends public key + info to CA.
CRLCertificate Revocation List. Periodic list of revoked certs. Stale between publishes.
OCSPOnline Certificate Status Protocol. Real-time revocation check. Better than CRL.
OCSP StaplingServer caches OCSP response and presents it. Reduces OCSP server load.
Wildcard cert*.domain.com — covers all subdomains. One compromise = all exposed.
SAN certSubject Alternative Name. Lists specific domains. More secure than wildcard.

Zero Trust vs Traditional — Core Difference
Traditional (perimeter/castle-and-moat): Trust everything inside the network. Firewall at the edge. Once you're in, you're trusted.

Zero Trust: Never trust, always verify. Every request authenticated regardless of network location. Control plane (policy engine/PA) evaluates every access decision. Data plane enforces it. Adaptive identity — trust score changes with behavior/context.
Zero Trust ComponentWhat It Does
Policy Engine (PE)Brain. Evaluates access requests against policy, user risk, device health.
Policy Administrator (PA)Issues/revokes session tokens based on PE decision.
Policy Enforcement Point (PEP)Gatekeeper. Allows or blocks traffic based on PA token.
Micro-segmentationEast-west traffic controlled granularly. No implicit trust between workloads.
Adaptive identityTrust score adjusts dynamically — anomalous behavior triggers step-up auth.
